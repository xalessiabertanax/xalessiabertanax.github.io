Case Study

Neural Pac-Man

A BCI-driven game that adapts difficulty dynamically based on cognitive workload measured via EEG

Role: BCI Logic Developer
Year: 2023
Duration: 3 months
PythonEEG (Unicorn Hybrid Black)UDPUX Research

01

The Challenge

Most video games have static difficulty levels. We wanted to create a closed-loop system where the game adapts itself based on how hard the player's brain is actually working, keeping them in a state of Flow.

Key Pain Points

1Static game difficulty leads to boredom or frustration
2No existing consumer games use real-time cognitive workload adaptation
3EEG signal noise (muscle movement, artifacts) makes real-time analysis challenging
4Need to balance adaptation speed with measurement reliability

02

The Process

01

Signal Acquisition

Used the Unicorn Hybrid Black headset to capture live EEG data. The challenge was filtering out noise to get a clean signal of mental workload.

02

Python Bridge

Wrote a Python middleware that received UDP data packets from the headset, calculated Alpha/Beta wave ratios in real-time to determine a 'Workload Score.'

03

Engagement Index

Implemented the Engagement Index formula (β / (α + θ)) based on EEG literature, with baseline calibration per player.

04

Adaptive Game Loop

Based on workload scores: High Load triggers 'Bullet Time' (ghosts slow down), Low Load triggers 'Chaos Mode' (ghosts speed up). Music tempo also adapts.

03

The Solution

A working Pac-Man prototype where game speed, ghost behavior, and music dynamically adapt to the player's real-time cognitive state measured via EEG.

✓Real-time EEG-based cognitive workload measurement
✓Engagement Index calculation from Alpha, Beta, and Theta waves
✓'Bullet Time' and 'Chaos Mode' adaptive game mechanics
✓Dynamic music tempo adaptation based on brain activity
✓Per-player baseline calibration system

04

The Impact

Successfully demonstrated that BCI can create deeper, more responsive entertainment experiences. Players could physically feel the game reacting to their focus levels.
